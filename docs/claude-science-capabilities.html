01

Agent architecture & orchestration

A coordinating generalist that plans, delegates to sub-agents, and audits itself.

  • Four bundled agent profiles INSTALL
    OPERON (main generalist — "compute, don't confabulate"; cites tool-returned IDs, never training memory), REVIEWER (mid-session fact-checker), BOOKMARKER (breadcrumb quotes for resuming), ONBOARDING (first-run interview → proposes 3 first tasks).
  • Plan → approve → execute KEYNOTEOURS
    One sentence yields a multi-phase plan with a confidence estimate; the user approves or iterates before execution. We saw "6 steps → Plan approved."
  • Natively multi-agent / sub-agents KEYNOTEDOCSOURS
    Parallel sub-agents, each with its own brief + output schema + transcript. Keynote demoed 100 parallel sub-agents across 100 diseases.
  • Background Reviewer (actor–critic) DOCSINSTALLOURS
    An independent agent re-examines claims/artifacts at checkpoints — results actually computed, value-vs-source consistency, citation/DOI accuracy, plan completion. Records "findings" on each artifact's Review tab; does not re-run analyses.
  • Scheduled / routine agent runs INSTALL — a first-class DB concept (routine scheduling, verification tables).
Our data corrects the press

The Reviewer is a genuinely separate model

Several outlets claim the reviewer runs on the same model as the work it checks (a criticism). Our first-hand run OURS plus the separate REVIEWER agent profile INSTALL show it's a distinct agent — we observed it running Sonnet 5 while the primary ran Opus 4.8.

04

Compute management

Runs where your data lives; scales one GPU to hundreds. DOCSKEYNOTE

Local (laptop / Linux / HPC login)SSH hostsSLURM (sbatch)Modal cloudNVIDIA BioNeMo NIM
  • Auto-reads ~/.ssh/config, probes CPU / GPU / CUDA / schedulers. Keynote demoed 2,200 compounds across 80 GPUs, fault-tolerant (2 failed, "no matter"), then a second independent model filtered survivors — a built-in falsification step.
  • Modal limits: 1 GiB input / 5 GiB output per job · 12h default (23h max) · 10 concurrent · no spend ceiling. Local GPU requires explicit enable (reduces sandboxing).
05

Environment & code execution

Python, R, shell in persistent kernels. DOCSINSTALL

Variables/dataframes survive across cells until ~30 min idle. A key install finding: base envs are deliberately lean (base python 3.11 = numpy/pandas/scipy/matplotlib/sklearn; r = tidyverse) and specialized packages install per-workspace on demand via a uv-managed .venv — so GPU/bio stacks (torch, scanpy, ESMFold…) never pollute the base image. Sources: conda (micromamba: conda-forge / bioconda / pytorch), pip/PyPI, CRAN/Bioconductor. Can compile from source in-sandbox and preserve builds as reusable artifacts.

06

Artifacts, provenance & reproducibility

"Reproducible by construction" — the platform's core pillar. DOCSKEYNOTEOURS

  • Artifact types — figures, datasets, reports, notebooks, 3D protein structures, genome-browser tracks, chemical structures, MSAs, HTML dashboards, and manuscripts (Markdown / LaTeX with \ref/\eqref resolution).
  • Immutable versioning — re-saving a filename makes a new version; each carries a 5-tab provenance panel: Messages · Code · Execution Log (authoritative) · Environment · Review.
  • Vision-grounded annotations — pin a comment directly on a figure / PDF / report and Claude "sees" it (no describing location in prose).
  • Local-first storage — artifacts & history live only in ~/.claude-science on the device; import/export up to 100 GB.

08

Safety, sandbox & permissions

Everything runs in an OS sandbox; every reach beyond it asks first. DOCSINSTALLOURS

  • OS sandbox (bubblewrap) — code reads/writes only the workspace + granted folders. We verified it runs on a managed/endpoint-secured corporate laptop.
  • Permission "approval cards" for folder access, code execution, network hosts, connector tools, remote jobs — scopes Once / This conversation / This project / Global, all revocable under Settings › Permissions. (We auto-approved these headlessly.)
  • Network allowlist of permitted hosts · Memory local-only, off by default · dangerous flags --dangerously-no-sandbox, --dangerously-skip-approvals. Only prompt/response API traffic leaves the device.

12

Limitations & critiques

Balance — from official docs & third parties

Not for clinical/diagnostic use; verify all results. Reviewer reduces but doesn't eliminate errors (checks claims, doesn't re-run). Residual hallucination ~5–15% reported by hands-on users 3P. Life-sciences only at launch. Beta gaps: no audit log, HIPAA not BAA-covered, no Modal spend ceiling. A rival OpenAI benchmark (GeneBench-Pro) reportedly scored Opus 4.8 ~16% vs GPT-5.6 ~29–31% 3P — adversarial, single source; cite with caution.
